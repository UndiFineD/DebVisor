# ISO Build & Installer Integration\n\n## Build Host Prerequisites\n\nA recent Debian-based build host is recommended; ideally match the target release (default: bookworm).\n    sudo apt update\n    sudo apt install -y live-build debootstrap xorriso squashfs-tools git\n\n## Live-Build Configuration\n\nDefined via `build/build-debvisor.sh`and`config/auto/config` (script form). Key flags:\n\n- `--distribution bookworm`\n\n- Hybrid ISO (`--binary-images iso-hybrid`)\n\n- Installer integration (`--debian-installer live`)\n\n- Firmware enabled (`--firmware-chroot true --firmware-binary true`)\n\n## Package Manifest\n\nLocated at `config/package-lists/debvisor.list.chroot` - single source of truth for included packages.\n\n- Core hypervisor, storage, and Kubernetes components are enabled by default.\n\n- Optional stacks (RPC service, web panel, VNC console, monitoring) are present but commented out and are installed via addons rather than baked into the ISO.\n\n## Installer Preseed\n\n`config/preseed.cfg` is intentionally thin. It seeds locale, hostname, disk recipe, and invokes a curses profile menu:\n\n- Writes profile choice to `/target/etc/debvisor-profile`(`ceph`,`zfs`, or`mixed`).\n\n- Keeps OS on first disk; remaining disks are left untouched for first-boot provisioning.\n\n- No complex late-commands; all smart logic happens at first boot.\n\nFor safety:\n\n- Any placeholder or lab-only passwords in `preseed.cfg`**must**be\n\n   overridden for real deployments. Treat the shipped file as an\n   example, not production-ready credentials.\n\n- For environments that require unattended installs, prefer keeping\n\n   secrets in a private include file (for example\n   `config/preseed.private.cfg` ignored by Git) and sourcing it from\n   the main preseed, or inject sensitive values via CI at build time\n   rather than committing them into this repository.\n\n## Hooks\n\nPlaced under `config/hooks/normal/*.sh` (executed inside chroot build):\n\n- Ensure ZFS DKMS builds\n\n- Preinstall Ceph stack\n\n- Preinstall Kubernetes components\n\n- Cockpit console modules\n\n## First Boot Provisioning\n\nSystemd unit: `debvisor-firstboot.service`\nScript: `debvisor-firstboot.sh`\nBehavior:\n\n1. Loads profile from `/etc/debvisor-profile` (ceph|zfs|mixed).\n\n1. Loads mode from `/etc/debvisor-mode`(`lab`or`prod`, default`lab`).\n\n1. Initializes logging under `/var/log/debvisor/*.status` (ceph, zfs, kubernetes, addons, etc.).\n\n1. Ensures base services (cockpit, libvirtd, KVM modules) and locale/time.\n\n1. Creates accounts depending on mode:\n\n- `lab`: ensures`root`,`node`,`monitor` exist; non-root accounts are locked by default.\n\n- `prod`: only ensures`root`; additional accounts are left to your workflow.\n\n1. Configures bridge networking and libvirt default storage.\n\n1. Detects disks; reserves first disk for OS; treats others as candidate storage disks.\n\n1. Provisioning is idempotent and guarded:\n\n- Ceph: skipped if `ceph status` already works; avoids wiping disks with existing data.\n\n- ZFS: skipped if `zpool list tank` succeeds; avoids wiping disks with existing data.\n\n- Kubernetes: skipped if `/etc/kubernetes/admin.conf` already exists.\n\n1. Configures Docker, initializes single-node Kubernetes + Calico, and applies ufw firewall rules (logging the active mode).\n\n1. Runs addons via Ansible (see below) and disables the first-boot service.\n\nIf any major step fails, a corresponding `*.status`marker under`/var/log/debvisor/`is set to`failed` for easy diagnosis.\n\n## Addons and Modes\n\n- **Mode control**: `/etc/debvisor-mode`\n\n- `lab`(default): convenience defaults, auto-create`node`/`monitor` users (locked), suitable for labs.\n\n- `prod`: minimal user changes; better suited for hardened environments.\n\n- **Addons config**:\n\n- Global flag file: `/etc/debvisor-addons.conf` with keys:\n\n- `ADDON_RPC_SERVICE`,`ADDON_WEB_PANEL`,`ADDON_VNC_CONSOLE`,`ADDON_MONITORING_STACK`(`yes`/`no`).\n\n- Profile defaults: `/etc/debvisor-addons.d/.conf`(e.g.`ceph.conf`,`zfs.conf`,`mixed.conf`).\n\n- On first boot, if `/etc/debvisor-addons.conf`is missing, a profile default is copied from`/etc/debvisor-addons.d/` when available.\n\n- **Addons execution**:\n\n- Ansible playbook: `ansible/playbooks/bootstrap-addons.yml`(mirrored into the image at`/usr/local/share/debvisor/ansible/bootstrap-addons.yml`).\n\n- First boot will run this playbook locally (if Ansible and the playbook are present), which conditionally applies roles:\n\n- `rpc-service`,`web-panel`,`vnc-console`,`monitoring-stack`.\n\n- Roles currently ship as stubs that create `/opt/debvisor-*/` placeholders and README notes; extend them to deploy real services.\n\n## Build Flow\n\n    ./build/build-debvisor.sh   # cleans, configures, builds\n    ls -lh live-image-amd64.hybrid.iso\nAs part of the build, `build/build-debvisor.sh` runs\n`build/sync-addons-playbook.sh` to mirror\n`ansible/playbooks/bootstrap-addons.yml` into\n`config/includes.chroot/usr/local/share/debvisor/ansible/`. This keeps\nthe in-image addons playbook aligned with the repository version\nwithout requiring manual copies.\n\n## Test Flow (VM or Bare Metal)\n\n1. Boot ISO\n\n1. Follow minimal installer prompts (locale, passwords, profile)\n\n1. Reboot -> first boot provisioning executes\n\n1. Validate:\n\n- Cockpit: <https://host:9090->>>>>>>>>>>>>>>>>>>>>>>>>> `ceph -s` healthy (ceph/mixed)\n\n- `zpool status` (zfs/mixed)\n\n- `kubectl get nodes` Ready\n\n## Extending\n\n- Add Helm charts under `docker\addons\`\n\n- Add cluster join script under `scripts/`\n\n- Pin versions via custom APT repo file under `config/includes.chroot/etc/apt/sources.list.d/`\n\n- Use `/etc/debvisor-addons.conf` and addons roles to introduce new optional stacks without changing the ISO package list.\n\n## Troubleshooting\n\n- Inspect build logs: `live-build.log`\n\n- Chroot failures: verify hook script shebangs and executable bit\n\n- Missing modules: confirm kernel flavor matches architecture (`amd64`)\n\n
