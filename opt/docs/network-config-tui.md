# Network Configuration TUI\n\nA curses-based terminal UI for configuring network interfaces on Linux hosts. It supports a single-bridge-by-default design (e.g., `br0`) with STP enabled, optional bonding, VLAN subinterfaces, and either systemd-networkd or Netplan backends.\n\n## Features\n\n- Single bridge: enslaves interfaces into one bridge (default `br0`) and configures IP on the bridge\n\n- STP enabled by default; adjustable timers (ForwardDelay, HelloTime, MaxAge)\n\n- Bonding: `bond0`with common modes (active-backup, 802.3ad, balance-xor, broadcast, balance-tlb, balance-alb)\n\n- Interfaces: wired, wireless (SSID/PSK), InfiniBand (basic IP)\n\n- VLANs: per-interface VLAN subinterfaces (e.g.,`eth0.100`)\n\n- Backends: generate systemd-networkd files or a single Netplan YAML\n\n- Apply script: `apply.sh`generated for safe, repeatable rollout\n\n## Requirements\n\n- Python 3.8+\n\n- Linux target for detection and application\n\n- Backend prerequisites:\n\n- systemd-networkd (if using`--backend networkd`)\n\n- netplan.io (if using `--backend netplan`)\n\n## Quick Start\n\n- Networkd backend (default):\n\n    python3 tools/netcfg-tui/netcfg_tui.py --output-dir ./out-networkd\n\n- Netplan backend:\n\n    python3 tools/netcfg-tui/netcfg_tui.py --backend netplan --output-dir ./out-netplan\n\n## TUI Controls\n\n- Up/Down or j/k: navigate\n\n- e: edit selected entry (bridge, bond, or interface)\n\n- s: save files to the output directory and generate `apply.sh`\n\n- r: reload interface list\n\n- q: quit\n\n## Generated Files\n\n- Networkd backend:\n\n- `10-br0.netdev`and`10-br0.network`(when single-bridge is enabled)\n\n-`10-.network`for each member (with`Bridge=br0`or`Bond=bond0`), and`10-..netdev`if VLAN set\n\n-`wpa_supplicant/wpa_supplicant-.conf`when Wi?Fi SSID/PSK provided\n\n- Netplan backend:\n\n-`99-debvisor.yaml`\n\n- Apply helper:\n\n- `apply.sh`to copy files into place and restart services (review before running with sudo)\n\n## Apply on Target\n\n- Networkd:\n\n    ./out-networkd/apply.sh\n\n- Netplan:\n\n    ./out-netplan/apply.sh\n\n## Notes & Limitations\n\n- Wi?Fi in client (station) mode generally cannot participate in a true L2 bridge due to 802.11/driver constraints; use routed/NAT or AP mode when needed\n\n- InfiniBand support covers basic IP; advanced partitioning (P_Key) is not included\n\n- Bond members can be auto-selected (all wired) or specified by name\n\n- The tool writes to an output directory and does not modify the system until you apply\n\n## Location\n\n- TUI script:`tools/netcfg-tui/netcfg_tui.py`\n\n- Documentation: this file (`docs/network-config-tui.md`)\n\n
