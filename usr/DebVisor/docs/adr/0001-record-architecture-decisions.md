# 1. Record Architecture Decisions

- Status: Accepted

- Deciders: DebVisor Team

- Date: 2025-12-06

## Context and Problem Statement

We want to record architectural decisions for future reference and onboarding.

## Decision Drivers

- Need for historical context on decisions.

- Need to understand trade-offs made in the past.

## Considered Options

- Wiki

- Jira tickets

- Markdown files in repository (ADRs)

## Decision Outcome

Chosen option: "Markdown files in repository (ADRs)", because it keeps the documentation close to the code and version controlled.

### Positive Consequences

- Version controlled history.

- Close to code.

### Negative Consequences

- Requires discipline to maintain.
