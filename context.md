# DebVisor Context\n\n## Structure\n\nTab size is 4 spaces\n\n## Overall Purpose\n\nDebVisor is a Debian 13-based mini hyper?converged hypervisor distro.\nIt is**containers?first**(Docker/Kubernetes), with VMs supported for\nlegacy and special?case workloads. The goal is a turnkey ISO that you\nboot, install via a curses (ncurses) installer, and end up with:\nKVM/libvirt + bridges + Cockpit web UI\nCephFS?first (with optional ZFS or mixed Ceph+ZFS)\nDocker + Kubernetes (kubeadm, containerd, Calico)\nOpinionated defaults and automated first?boot provisioning.\nKey entry points:\nREADME.md - high?level description, profiles, and ISO build quickstart.\nMakefile + build-debvisor.sh - one?shot live-build wrapper to produce the ISO.\nDebVisor_initial.md - now just a stub table listing canonical file locations.\n\n## Installer & Image Build\n\nLive-build config:\npreseed.cfg - Debian Installer preseed with:\nLocale/timezone (en_US, UTC).\nHostname debvisor.local.\nPrompts for root + admin passwords.\nCustom profile menu (ceph | zfs | mixed) via whiptail, writing PROFILE=... to /etc/debvisor-profile.\nLate command creating node and monitor accounts with interactive passwords.\ndebvisor.list.chroot - APT package manifest installing:\nBase tools (sudo, ssh, nftables, ufw, fail2ban, tmux, nano, mc, etc.).\nVirtualization (qemu-kvm, libvirt, virt-manager, virt-viewer, guest agent, SPICE).\nWeb management (Cockpit + submodules, nginx?light).\nStorage (Ceph MON/MGR/OSD/MDS, RBD, CephFS; ZFS; LVM/RAID).\nContainers + K8s (docker.io, containerd, kubeadm/kubelet/kubectl).\nRPC/UI groundwork (python3?grpcio/zmq, Flask, gunicorn, node exporter, noVNC/websockify, tigervnc).\nBuild tooling:\nMakefile - wraps live-build with DebVisor settings (bookworm, ISO?hybrid, preseed, firmware, mirrors).\nbuild-debvisor.sh - script that runs live-build using the above config, with environment flags for `DEBVISOR_DIST`(Debian release, default`bookworm`),`DEBVISOR_ARCH`(architecture, default`amd64`; tested primarily on amd64, with arm64/riscv64 as advanced/experimental targets),`DEBVISOR_FAST`(skip`lb clean`for quicker rebuilds),`DEBVISOR_VERSION`(tags the ISO filename), mirror/firmware toggles, and a`DEBVISOR_SELFTEST`mode that runs preflight +`lb config` only for CI.\nResult: a `live-image-.hybrid.iso`(or`debvisor--.hybrid.iso`) that boots into the DebVisor installer.\n\n## First?Boot Provisioning & System Services\n\nMain provisioning script:\ndebvisor-firstboot.sh:\nReads PROFILE from /etc/debvisor-profile (default ceph).\nEnsures Cockpit and libvirt are enabled; basic locale/time setup.\nVerifies root, node, monitor users (creates/locks non?root until passwords set).\nNetworking: creates br0, attaches the primary NIC, brings bridge up.\nDisk detection: identifies OS disk vs. "extra" storage disks for Ceph/ZFS.\nCeph path (ceph/mixed):\nGenerates ceph.conf, bootstraps MON/MGR/MDS.\nWipes extra disks, creates OSDs via ceph-volume lvm.\nCreates RBD pool and a CephFS filesystem, mounts CephFS at /srv/cephfs via /etc/fstab.\nZFS path (zfs/mixed):\nWipes extra disks, creates tank pool, enables lz4, datasets tank/vm, tank/docker, tank/k8s mounted under /srv/*.\nConfigures Docker (daemon.json) and enables it.\nInitializes single?node Kubernetes (swapoff, containerd, kubeadm init, Calico CNI, removes control-plane taint).\nConfigures ufw firewall (allow 22, 9090, 6443).\nDefines and autostarts libvirt default storage pool.\nDisables its own systemd unit after successful run.\nSystemd unit (not re-read here but referenced from README):\ndebvisor-firstboot.service - runs the script once on first boot, sources /etc/debvisor-profile.\n\n## Core Configuration & Addons\n\nconfig:\npreseed.cfg, package-lists/, and live-build hooks/normal/*.sh (Ceph, ZFS, K8s, Cockpit).\nincludes.chroot/... -
